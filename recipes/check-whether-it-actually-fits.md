# Recipe: Check Whether It Actually Fits

One job, one page. Everything you need is here; nothing else in the repository is required to use this.

| | |
| --- | --- |
| **Helps with** | Working out where an offer is a good fit, a poor fit, or still uncertain for a specific prospect, before building a business case around a use case that was never going to work |
| **You need** | Every use case the prospect actually described, team by team or role by role, and confirmed product capability, not what seems like it should probably work |
| **You'll get** | Each use case classified as a good fit, a poor fit, or genuinely uncertain, with the specific reason stated plainly rather than softened |
| **Open** | [Workflow](../workflows/13-fit-and-limitations-review.md) · [Prompt](../templates/fit-and-limitations-review-prompt.md) · [Skill](../.agents/skills/fit-and-limitations-review/SKILL.md) · [Worked example](../examples/kellow-fit-review-output.md) |
| **The AI cannot decide** | What your product is actually confirmed to do when that itself is unclear, or what to actually build a business case around, disqualify, or tell the prospect |
| **You must check** | Every good fit traces to something actually confirmed, not an assumed capability; every poor fit is a real, named mismatch, never reframed as a hidden advantage; "uncertain" is used honestly, not as a softer way of saying no |
| **Then** | Decide yourself which use cases to build a case around, which to disqualify, and which need more evidence before either |

Want the fuller method or how to classify a mixed case honestly? Open the [workflow](../workflows/13-fit-and-limitations-review.md) itself.
