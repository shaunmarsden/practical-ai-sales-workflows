# Fictional Objection Pattern Log

> This is an entirely fictional log, created to test the Objection Pattern Review workflow. It reuses two objections already on record from the Hartwell Analytics and Bramfield Insurance Group stories, and adds four fresh fictional entries (Marlow Consulting, Ashcombe Underwriting, Drayton Manufacturing) to give a genuine pattern something real to work with. Every new company, person and detail is invented.

The point of this log is that it contains one real, useful pattern and one misleading one that only looks like a pattern until it is checked properly.

## Deliberate Test Points

- **The Copilot pattern is a trap.** Three entries (1, 4, 5) mention Copilot, which could look like a single "we are losing to Copilot" problem if only counted, not read. The actual drivers are three different things: a genuine tooling comparison with a budget-timing element (1), a budget constraint that had nothing to do with tool preference (4), and a real, final, company-wide standardisation with no route back (5). Only one of the three (5) was actually lost, and it was lost for a reason unrelated to the product's capability. Treating "Copilot mentioned three times" as one systemic issue would misdiagnose all three.
- **The compliance pattern is real.** Two unrelated entries (3, 6), in different sectors, both raise a specific, similar, answerable question about what happens to AI-generated outputs derived from regulated or audited source material. This is a genuine recurring pattern worth acting on, not a coincidence of wording.
- **Confidence should differ by finding.** The compliance pattern is high-confidence: the underlying need is essentially identical each time. The Copilot "pattern" is low-confidence as a systemic finding, precisely because the surface wording is similar but the underlying drivers are not; the honest finding is that there is no single Copilot problem, not that there is one.
- **Do not average the outcomes.** One loss out of three Copilot-adjacent entries is not "a 33% loss rate to Copilot"; entry 5 was a standardisation decision made before the conversation even started, not a comparison this deal could have won.
